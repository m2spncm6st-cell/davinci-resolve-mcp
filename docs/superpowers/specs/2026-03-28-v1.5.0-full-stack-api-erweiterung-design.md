# v1.5.0 — Full Stack API-Erweiterung

## Ziel

Alle sinnvollen, noch ungenutzten Methoden der DaVinci Resolve Scripting API in die bestehenden MCP-Tools integrieren. Kein neues Tool — 22 neue Actions in 4 bestehende Tools. Ergebnis: 11 Tools, 111+ Actions.

## Architektur-Entscheidungen

- **Hybrid-Integration**: Neue Actions in bestehende Tools, wo sie thematisch passen
- **Kein Stereo/3D**: YAGNI — zu nischig, kann bei Bedarf nachgerüstet werden
- **Kein Clip-Trimming**: API bietet keine Setter für Start/End/Duration — nur indirekt über Splits/Deletes möglich (bereits in v1.4.0 via Marker-basiertes Editing gelöst)
- **Bestehende Patterns beibehalten**: safe_tool, Navigation-Helpers, _err/_ok/_ser, Compound-Tool-Pattern

## 1. Timeline — 2 neue Actions

### get_timecode

Playhead-Position auslesen.

- **API:** `Timeline.GetCurrentTimecode()`
- **Parameter:** keine zusätzlichen
- **Response:** `_ok(timecode="01:00:10:00")`

### set_timecode

Playhead an bestimmte Position setzen.

- **API:** `Timeline.SetCurrentTimecode(timecode)`
- **Parameter:** `name` (str) — Timecode-String im Format `"HH:MM:SS:FF"`
- **Validierung:** Format-Check vor API-Call
- **Response:** `_ok(timecode="01:00:10:00")`
- **Fehler:** `_err("Invalid timecode format")` bei ungültigem Input

## 2. Timeline Item — 10 neue Actions

### Takes (5 Actions)

#### get_takes

Alle Takes eines Clips auflisten.

- **API:** `TimelineItem.GetTakesCount()` + `TimelineItem.GetTakeByIndex(idx)` für jeden Take
- **Parameter:** `track_type`, `track_index`, `item_index` (wie bestehend)
- **Response:** `_ok(count=3, takes=[{index: 1, ...}, ...])`

#### get_selected_take

Aktuell aktiven Take abfragen.

- **API:** `TimelineItem.GetSelectedTakeIndex()`
- **Parameter:** `track_type`, `track_index`, `item_index`
- **Response:** `_ok(selected_take=2)`

#### add_take

Alternativen Clip als Take hinzufügen.

- **API:** `TimelineItem.AddTake(mediaPoolItem, startFrame, endFrame)`
- **Neue Parameter:** `media_pool_clip` (str — Clip-Name im Media Pool), `start_frame` (int), `end_frame` (int)
- **Logik:** Clip-Name im aktuellen Media-Pool-Ordner suchen, MediaPoolItem-Objekt holen
- **Response:** `_ok(added=True, clip="ClipName.mp4")`
- **Fehler:** `_err("Clip 'X' not found in current Media Pool folder")`

#### select_take

Zu einem Take wechseln.

- **API:** `TimelineItem.SelectTakeByIndex(idx)`
- **Neuer Parameter:** `take_index` (int, 1-basiert)
- **Response:** `_ok(selected=True, take_index=2)`

#### delete_take

Take entfernen.

- **API:** `TimelineItem.DeleteTakeByIndex(idx)`
- **Parameter:** `take_index` (int, 1-basiert)
- **Response:** `_ok(deleted=True, take_index=2)`

### finalize_take

Aktiven Take festschreiben.

- **API:** `TimelineItem.FinalizeTake()`
- **Parameter:** `track_type`, `track_index`, `item_index`
- **Response:** `_ok(finalized=True)`

### Cache-Steuerung (2 Actions)

#### get_cache

Cache-Status eines Clips abfragen.

- **API:** `TimelineItem.GetIsColorOutputCacheEnabled()` + `TimelineItem.GetIsFusionOutputCacheEnabled()`
- **Parameter:** `track_type`, `track_index`, `item_index`
- **Response:** `_ok(color_cache=True, fusion_cache=False)`

#### set_cache

Cache aktivieren/deaktivieren.

- **API:** `TimelineItem.SetColorOutputCache(value)` oder `TimelineItem.SetFusionOutputCache(value)`
- **Neuer Parameter:** `cache_type` (str: `"color"` oder `"fusion"`), `property_value` (str: `"true"` / `"false"`)
- **Validierung:** `cache_type` muss `"color"` oder `"fusion"` sein
- **Response:** `_ok(cache_type="color", enabled=True)`

### update_sidecar

Sidecar-Daten für BRAW/R3D aktualisieren.

- **API:** `TimelineItem.UpdateSidecar()`
- **Parameter:** `track_type`, `track_index`, `item_index`
- **Response:** `_ok(updated=True)`

### stabilize

Video-Stabilisierung auf Clip anwenden.

- **API:** `TimelineItem.Stabilize()`
- **Parameter:** `track_type`, `track_index`, `item_index`
- **Response:** `_ok(stabilized=True)`
- **Hinweis:** Kann lange dauern — Resolve analysiert den Clip

## 3. Color — 7 neue Actions

### Color Versions (5 Actions)

#### list_versions

Alle Versions eines Clips auflisten.

- **API:** `TimelineItem.GetVersionNameList(versionType)`
- **Neuer Parameter:** `version_type` (int: 0=local, 1=remote; default 0)
- **Response:** `_ok(versions=["Version 1", "Version 2"], version_type=0)`

#### get_version

Aktive Version abfragen.

- **API:** `TimelineItem.GetCurrentVersion()`
- **Response:** `_ok(version={"name": "Version 1", "type": 0})`

#### add_version

Neue Color Version erstellen.

- **API:** `TimelineItem.AddVersion(versionName, versionType)`
- **Neuer Parameter:** `version_name` (str), `version_type` (int: 0=local, 1=remote; default 0)
- **Response:** `_ok(added=True, version_name="Warm Look", version_type=0)`

#### load_version

Zu einer Version wechseln.

- **API:** `TimelineItem.LoadVersionByName(versionName, versionType)`
- **Parameter:** `version_name`, `version_type`
- **Response:** `_ok(loaded=True, version_name="Warm Look")`

#### delete_version

Version löschen.

- **API:** `TimelineItem.DeleteVersionByName(versionName, versionType)`
- **Parameter:** `version_name`, `version_type`
- **Response:** `_ok(deleted=True, version_name="Warm Look")`
- **Fehler:** `_err("Cannot delete the only remaining version")`

### Magic Mask (2 Actions)

#### create_magic_mask

Magic Mask auf aktuellem Clip erstellen.

- **API:** `TimelineItem.CreateMagicMask(mode)`
- **Neuer Parameter:** `mode` (str: `"F"` = Forward, `"B"` = Backward, `"BI"` = Bidirectional)
- **Mapping:** `mode` wird über den `lut_path`-Parameter übergeben (freier String-Param im color-Tool)
- **Response:** `_ok(created=True, mode="F")`
- **Hinweis:** Kann lange dauern — Resolve analysiert den Clip

#### regenerate_magic_mask

Bestehende Magic Mask neu berechnen.

- **API:** `TimelineItem.RegenerateMagicMask()`
- **Response:** `_ok(regenerated=True)`

### LUT Export (1 Action)

#### export_lut

LUT aus aktuellem Clip exportieren.

- **API:** `TimelineItem.ExportLUT(exportType, path)`
- **Parameter:** `node_index` (int — exportType: 0=.cube, 1=.3dl), `lut_path` (str — Zielpfad)
- **Validierung:** Pfad muss mit `.cube` oder `.3dl` enden, exportType muss 0 oder 1 sein
- **Response:** `_ok(exported=True, path="/path/to/output.cube")`

## 4. Fairlight — 3 neue Actions

### get_voice_isolation

Voice-Isolation-Status abfragen.

- **API:** `TimelineItem.GetVoiceIsolationState()`
- **Parameter:** `track_index`, `item_index`
- **Logik:** Audio-Track-Item holen (wie bei get_volume/set_volume)
- **Response:** `_ok(enabled=True, amount=75)`

### set_voice_isolation

Voice Isolation konfigurieren.

- **API:** `TimelineItem.SetVoiceIsolationState({"enabled": bool, "amount": int})`
- **Neue Parameter:** `enabled` (bool), `amount` (int, 0-100)
- **Validierung:** `amount` muss zwischen 0 und 100 liegen
- **Response:** `_ok(enabled=True, amount=75)`

### get_audio_mapping

Audio-Channel-Mapping eines Clips auslesen.

- **API:** `TimelineItem.GetSourceAudioChannelMapping()`
- **Parameter:** `track_index`, `item_index`
- **Response:** `_ok(mapping={...})` — JSON-Struktur von Resolve

## Testing-Strategie

- **Unit-Tests** (ohne Resolve): Response-Format, Parameter-Validierung, Fehlerbehandlung
- **Integration-Tests** (mit Resolve): Alle neuen Actions gegen laufendes Resolve
- **Bestehende Tests dürfen nicht brechen** — reine Erweiterung, keine Breaking Changes

## Nicht enthalten (bewusst ausgeschlossen)

- **Stereo/3D-Support**: Zu nischig (YAGNI)
- **Clip-Trimming (SetStart/SetEnd)**: API bietet keine Setter — bereits via Marker-basiertem Editing in v1.4.0 gelöst
- **Keyframes**: API bietet keine Keyframe-Methoden für Timeline-Items
- **Rename Version**: Könnte nachgerüstet werden, aber selten benötigt
